Introduction

The sinking of the Titanic is one of the most well-known disasters in history. On April 15, 1912, during its first voyage, the Titanic sank after hitting an iceberg, resulting in the deaths of 1,502 out of 2,224 passengers and crew members. This tragedy led to the implementation of better safety regulations for ships. One of the main reasons for the high number of casualties was the insufficient number of lifeboats for the passengers and crew. While luck played a part in survival, certain groups of people, such as women, children, and those from the upper class, were more likely to survive than others. This analysis aims to investigate the factors that influence survival rates. Specifically, we will use machine learning techniques to predict which passengers were more likely to survive the disaster.

Goal

This analysis aims to predict which passengers survived the sinking of the Titanic using machine learning techniques. The study will involve data cleaning, exploratory data analysis, feature engineering, and predictive modeling. The model's accuracy will be evaluated using the area under the receiver operating characteristic curve (AUC). The final model will be used to make predictions on the test dataset provided by Kaggle.

Methodology

The analysis was carried out using Python version 3.8.5. The data was cleaned and preprocessed by utilizing the Pandas library. Exploratory data analysis was conducted by implementing the Matplotlib and Seaborn libraries. Feature engineering was performed to generate new features that could enhance the model's predictive capability. Finally, the predictive model was built using the Scikit-learn and XGBoost libraries.

Results

The XGBoost model achieved a reasonable accuracy rate of 81%. However, there is room for further improvement by enhancing the model and doing more feature engineering. The final model was used to make predictions on the training dataset provided by Kaggle. However, the results have yet to be submitted to Kaggle or measured using their scoring system. Fine-tuning the model will be completed before submitting the results to Kaggle.

Data Dictionary

  • PassengerId: Identifier for each passenger.
  • Survived: Survival status (0 = No, 1 = Yes).
  • Pclass: Ticket class (1 = 1st, 2 = 2nd, 3 = 3rd).
  • Name: Name of the passenger.
  • Sex: Gender of the passenger.
  • Age: Age in years.
  • SibSp: Number of siblings/spouses aboard the Titanic.
  • Parch: Number of parents/children aboard the Titanic.
  • Ticket: Ticket number.
  • Fare: Passenger fare.
  • Cabin: Cabin number.
  • Embarked: Port of Embarkation (C = Cherbourg, Q = Queenstown, S = Southampton).

Missing Data

The dataset has missing values in three columns:

  • Age: 177 missing values
  • Cabin: 687 missing values
  • Embarked: 2 missing values

We'll handle these as follows:

  • Age: We can impute the missing ages. A common approach is to use the median age. Still, we could consider using the median age within specific groups (e.g., classified by class or gender) for a more accurate imputation.
  • Cabin: With 687 missing values out of 891 total entries, this column has significant missing data. We might consider dropping it or deriving a more straightforward feature, like whether a cabin was assigned.
  • Embarked: Only two missing values. We can fill these with the most common point of embarkation.

Let's proceed with this data wrangling. We'll also convert categorical variables like 'Sex' and 'Embarked' into numerical formats for easier analysis later on.

In[4]:
# Data cleaning and preparation

# Impute missing Age values with the median
titanic_data['Age'].fillna(titanic_data['Age'].median(), inplace=True)

# Convert 'Cabin' to a binary feature (1 if cabin is assigned, 0 otherwise)
titanic_data['Cabin'] = titanic_data['Cabin'].apply(lambda x: 0 if pd.isna(x) else 1)

# Impute missing 'Embarked' values with the most common port
most_common_port = titanic_data['Embarked'].mode()[0]
titanic_data['Embarked'].fillna(most_common_port, inplace=True)

# Convert 'Sex' into a numerical format (0 for male, 1 for female)
titanic_data['Sex'] = titanic_data['Sex'].map({'male': 0, 'female': 1})

# Re-check for missing values
missing_values_after = titanic_data.isnull().sum()
missing_values_after[missing_values_after > 0]

Logistic Regression Modeling

Now, let's move on to building a logistic regression model. Logistic regression is a good starting point for binary classification problems like this one. We'll first prepare our data for modeling, then fit a logistic regression model and evaluate its performance.

We'll start by selecting our features for the model and splitting the data into a training set and a test set. We'll use the 'Sex,' 'Age,' 'Pclass,' and 'Fare' features for our model.

In[9]:
from sklearn.model_selection import train_test_split
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import classification_report, confusion_matrix

# Selecting features and target variable
features = ['Sex', 'Age', 'Pclass', 'Fare']
X = titanic_data[features]
y = titanic_data['Survived']

# Splitting the data into training and test sets
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

# Create and train the logistic regression model
logreg = LogisticRegression()
logreg.fit(X_train, y_train)

# Predicting on the test set
y_pred = logreg.predict(X_test)

# Evaluating the model
conf_matrix = confusion_matrix(y_test, y_pred)
class_report = classification_report(y_test, y_pred)

Gradient-Boosting Modeling

Let's proceed with creating a gradient-boosting model, a sophisticated machine-learning technique that involves the construction of a robust predictive model by sequentially combining an ensemble of weak models. Gradient boosting is particularly effective in handling complex datasets and has been widely used in various fields, including finance, healthcare, and marketing. By leveraging the power of gradient boosting, we can obtain a high-performing model that accurately predicts outcomes and outperforms other traditional machine learning models.

In[10]:
import xgboost as xgb
from sklearn.metrics import accuracy_score
from sklearn.model_selection import train_test_split

# Selecting features and target variable
features = ['Sex', 'Age', 'Pclass', 'Fare']
X = titanic_data[features]
y = titanic_data['Survived']

# Splitting the data into training and test sets
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

# Creating the DMatrix for XGBoost
dtrain = xgb.DMatrix(X_train, label=y_train)
dtest = xgb.DMatrix(X_test, label=y_test)

# Setting the hyperparameters for XGBoost
params = {
    'objective': 'binary:logistic',
    'eval_metric': 'error'
}

# Training the XGBoost model
num_rounds = 100
model = xgb.train(params, dtrain, num_rounds)

# Making predictions on the test set
y_pred = model.predict(dtest)
y_pred_binary = [round(value) for value in y_pred]

# Evaluating the model
accuracy = accuracy_score(y_test, y_pred_binary)
accuracy
